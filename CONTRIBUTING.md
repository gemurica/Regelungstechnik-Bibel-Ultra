# Contributing

Danke fuer jeden Beitrag. Das Ziel ist eine faire, nachvollziehbare
Zusammenarbeit mit klarer Urheberschaft.

## Grundregeln
- Behandle alle Mitwirkenden respektvoll.
- Reiche Aenderungen ueber Pull Requests ein (kein direkter Push auf `main`).
- Halte Aenderungen fachlich korrekt und gut nachvollziehbar.

## Beitrag einreichen
1. Repo forken und einen Branch erstellen (`feature/...`, `fix/...`, `docs/...`).
2. Aenderungen committen (klare Commit-Nachrichten).
3. Pull Request gegen `main` erstellen und kurz beschreiben:
   - Was wurde geaendert?
   - Warum ist die Aenderung sinnvoll?
   - Welche Dateien/Abschnitte sind betroffen?

## Namensnennung (wichtig)
- Wer einen inhaltlichen Beitrag leistet, soll genannt werden.
- Fuege dich bei substanziellen Beitragen im `NOTICE` unter "Contributors" ein:
  `- Name (@github-handle) - kurzer Beitrag`
- Bei kleineren Korrekturen erfolgt Nennung ueber die Git-Historie.

## Lizenz der Beitraege
Mit dem Einreichen eines Beitrags bestaetigst du, dass:
- dein Beitrag von dir stammt oder rechtmaessig eingebracht wurde, und
- dein Beitrag unter derselben Lizenz wie dieses Repository steht
  (`CC BY-NC-SA 4.0`, siehe `LICENSE`).

## Review und Merge
- Pull Requests werden vor Merge fachlich geprueft.
- Merge erfolgt erst nach Freigabe durch den Maintainer.
